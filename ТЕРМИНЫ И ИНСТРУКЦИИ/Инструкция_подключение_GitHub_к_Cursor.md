# Подключение GitHub к Cursor — пошаговая инструкция

Простым языком: что делать по порядку и как пользоваться командами **push**, **pull** и остальными.

---

## Зачем это нужно (очень коротко)

- **Git** — программа на **твоём компьютере**. Она запоминает версии файлов в папке проекта: «сохранение с подписью».
- **GitHub** — **сайт в интернете**, куда эти версии **выкладывают**. Туда же заходят люди из команды.

**Аналогия:** Git — как история изменений в документе, но для всей папки. GitHub — общая папка в облаке для проекта.

---

## Шаг 1. Аккаунт на GitHub

1. Зайди на [https://github.com](https://github.com).
2. Зарегистрируйся (логин, почта, пароль).
3. Подтверди почту, если сайт попросит.

---

## Шаг 2. Установить Git на Windows (один раз)

1. Скачай установщик: [https://git-scm.com/download/win](https://git-scm.com/download/win).
2. Запускай установку, настройки по умолчанию обычно подходят — жми «Далее».
3. После установки **полностью закрой и снова открой Cursor**, чтобы он увидел Git.

---

## Шаг 3. Как открыть терминал в Cursor

Команды из инструкции вводятся **в терминале** — панель внизу окна с мигающим курсором.

**Способ 1 (быстрый):** нажми **`Ctrl + `** (клавиша с **обратной кавычкой** — обычно слева от цифры 1, на той же кнопке, что и «Ё»).

**Способ 2:** вверху меню **Terminal** → **New Terminal** (или **Терминал** → **Новый терминал**).

**Способ 3:** **`Ctrl + Shift + P`** → начни печатать `terminal` → выбери **Terminal: Create New Terminal** / **Создать новый терминал**.

Внизу может быть написано **PowerShell** или **Command Prompt** — это нормально.

---

## Шаг 4. Представиться Git’у (имя и почта) — две команды по очереди

Это делается **один раз** (или когда захочешь сменить данные).

**Не одна длинная строка.** Вводишь **первую** команду → нажимаешь **Enter** → потом **вторую** → снова **Enter**.

Подставь **свои** имя и почту (почта желательно **та же**, что на GitHub):

```text
git config --global user.name "Твоё Имя"
```

```text
git config --global user.email "твоя@почта.com"
```

После каждой команды часто **ничего не выводится** — это нормально, значит сработало.

**Проверка (по желанию):**

```text
git config --global --list
```

Должны быть строки с `user.name` и `user.email`.

---

## Шаг 5. Вход в GitHub с компьютера (ничего не ломай заранее)

Отдельный «секретный шаг» перед всем остальным **часто не нужен**.

Чаще всего ты **впервые входишь в аккаунт**, когда делаешь **`git push`** или **`git clone`**: откроется **браузер** — залогинься на GitHub и разреши доступ. Так работает вход на Windows.

Если терминал просит **логин и пароль** при `push`:

- **Username** — твой логин GitHub.
- **Password** — не обычный пароль от сайта, а **Personal Access Token** (длинный код из настроек GitHub: *Settings → Developer settings → Personal access tokens*). Его создают один раз и вставляют вместо пароля. **Никому не показывай токен.**

Если что-то не получилось — сохрани **точный текст ошибки** из терминала и разберись с ним по шагам или с помощью знакомых.

## Шаг 6. Создать репозиторий на GitHub

1. На GitHub нажми **+** → **New repository**.
2. Придумай **имя** репозитория (латиницей, без пробелов можно через `-`).
3. Можно создать **пустой** репозиторий **без** галочки README — так проще для первого раза.
4. Нажми **Create repository**.

Скопируй адрес репозитория, он выглядит так:

`https://github.com/твой-логин/имя-репозитория.git`

Он понадобится на следующем шаге.

## Шаг 7. В папке «ещё нет Git» — что вводить по порядку

**«Нет Git»** значит: ты ещё **не делала** `git init` в этой папке. Тогда команды идут **строго по очереди** — каждая **отдельно**, после каждой **Enter**.

### 7.1. Открой правильную папку

В Cursor: **File → Open Folder** и выбери **папку проекта**, которую хочешь связать с GitHub.

Терминал должен быть открыт **в этой папке** (обычно Cursor сам открывает терминал в корне открытой папки).

### 7.2. Команды подряд (подставь свой адрес в шаге с `remote`)

| № | Команда | Что делает |
|---|---------|------------|
| 1 | `git init` | Включает учёт версий в этой папке. |
| 2 | `git add .` | Добавляет все файлы в «корзину» для сохранения. |
| 3 | `git commit -m "Первое сохранение"` | Делает первый снимок с подписью. |
| 4 | `git branch -M main` | Называет ветку **main** (как обычно на GitHub). |
| 5 | `git remote add origin https://github.com/ТВОЙ-ЛОГИН/ИМЯ-РЕПО.git` | Привязывает твой проект к репозиторию на GitHub. |
| 6 | `git push -u origin main` | **Отправляет** файлы на GitHub. Здесь может попросить войти в аккаунт. |

Пример **только для пункта 5** (замени на свой URL):

```text
git remote add origin https://github.com/ivanova/moy-proekt.git
```

Если Git напишет, что **remote origin уже существует** — напиши, что именно показал терминал; возможно, привязка уже была сделана раньше.

### 7.3. Если на GitHub при создании репозитория ты поставила галочку README

На сайте уже есть файлы, которых нет у тебя в первом коммите. Тогда **перед** успешным первым `push` иногда нужно подтянуть их:

```text
git pull origin main --allow-unrelated-histories
```

Потом снова:

```text
git push -u origin main
```

Если откроется непонятный редактор — лучше спросить по ситуации или прислать скрин/текст.

Если репозиторий на GitHub **полностью пустой** — команду `pull` можно **не** делать.

## Шаг 8. Каждый день: как работать с изменениями

**Перед работой в команде** полезно забрать чужие правки:

```text
git pull
```

После того как что-то изменила в файлах:

```text
git add .
git commit -m "Коротко что сделала"
git push
```

**Простая схема:**

1. **pull** — забрать с GitHub то, что сделали другие.
2. Поработала локально.
3. **add → commit → push** — отправить свои изменения.

---

## Шаг 9. Подключиться к чужому проекту (команда)

Тебе дают ссылку на репозиторий. В терминале, в **удобной папке** (не обязательно внутри Cursor сначала):

```bash
git clone https://github.com/команда/проект.git
```

Потом в Cursor: **Open Folder** → выбери **скачанную папку**. Дальше — те же **pull**, **add**, **commit**, **push**, если тебе выдали **права на запись** в репозиторий.

---

## Шаг 10. То же самое без команд — панель в Cursor

Слева значок ветки / **Source Control** (часто как развилка). Там можно:

- видеть изменённые файлы;
- ввести сообщение коммита и нажать **Commit**;
- использовать **Sync** / **Push** / **Pull** (названия зависят от версии).

Терминал и кнопки делают **одно и то же** — выбирай, как удобнее.

---

## Короткий словарь

| Слово | Простыми словами |
|--------|------------------|
| **Репозиторий (repo)** | Проект на GitHub + история версий |
| **Commit** | Снимок «как сейчас» + короткая подпись |
| **Push** | Отправить свои коммиты на GitHub |
| **Pull** | Скачать с GitHub новые коммиты к себе |
| **Clone** | Первый раз скачать весь проект с GitHub |
