# Правила генерации постов FlyFIT (для агента)

Перед написанием поста **всегда** опирайся на актуальные данные из репозитория, а не только на общие знания о маркетинге.

## 1. Откуда брать «новые» данные

Читай и учитывай (по задаче — не обязательно всё каждый раз):

| Источник | Зачем |
|----------|--------|
| `О_БРЕНДЕ/Карточка_бренда_для_СММ_агента.md` | Роль бренда, тон, ЦА, запреты |
| `О_БРЕНДЕ/Позиционирование_бренда.md` | Архетипы, формулировки миссии, визуальный код |
| `О_БРЕНДЕ/Целевая аудитория.txt` | Сегменты, боли, доходы (если тема про сегмент) |
| `КОНТЕНТ/БАЗА/100постов_тг.csv` | **Образец тона, структуры, артикулов** — перечитай несколько релевантных строк |
| `КОНТЕНТ/БАЗА/Рубрики.md` | Если пост в рамках рубрики |
| `КОНТЕНТ/БАЗА/Контент-план.md` | Если нужно вписаться в план |
| `КОНТЕНТ/БАЗА/ГАЙДЫ/*.md` | Готовые гайды и чек-листы (отёки, рутина, челленджи) — факты и формулировки |
| `КОНТЕНТ/.cursorrules/.cursorrules` | Общие правила папки КОНТЕНТ |
| Файлы в `КОНТЕНТ/ПОСТЫ/`, `КОНТЕНТ/Reels/` | Текущие ТЗ и черновики по теме |

**Вовлечение / опросы / розыгрыши:** голос бренда тот же; механики — ориентир `КОНТЕНТ/БАЗА/Посты_конкуренты.csv` (только формат, не чужие бренды).

Если в задаче даны **новые факты** (акция, артикул, дата) — они **важнее** шаблона; не противоречь им.

## 2. Кто выбирает тему поста

**Если пользователь задал конкретную тему** (продукт, боль, рубрика, «про спину», «про валик мягкий», акция, дата) — пиши **только** про неё, без самовольной смены фокуса.

**Если тема не задана** — типовые запросы вроде «напиши пост в ТГ», «новый пост в канал», «сгенерируй пост» — **тему выбираешь ты сам.** Нельзя отвечать общими фразами вместо поста: нужен полноценный текст и осмысленная тема.

Как выбрать тему без явного ТЗ:

1. Опирайся на **`КОНТЕНТ/БАЗА/100постов_тг.csv`**: по возможности просмотри **несколько последних или разнотипных** постов, чтобы **не повторять** тот же угол, ту же мысль или тот же продукт подряд без необходимости (если в файле видно повторение — смени акцент: другая боль, другой инвентарь, другой формат подачи).
2. Сверься с **`КОНТЕНТ/БАЗА/Рубрики.md`** и при наличии с **`КОНТЕНТ/БАЗА/Контент-план.md`**: что логично «подтянуть» по рубрикам или плану, чего давно не было.
3. Удерживай **`О_БРЕНДЕ/Карточка_бренда_для_СММ_агента.md`**: ЦА «Марина», домашний фитнес, простота, забота — тема должна быть в этом контексте.
4. Допустимы смешение: экспертный совет + мягкое упоминание подходящего товара с артикулом, если уместно по стилю базы.

В начале ответа, **когда тему выбрал сам**, добавь короткий блок (2–4 строки):

```markdown
**Тема поста:** …
**Почему эта тема:** … (например: давно не было про … / дополняет недавние посты / подходит под рубрику … / закрывает боль …)
```

Затем сразу идёт пост в Markdown по шаблону из §4.

## 3. Стиль текста (кратко)

- Тёплый, на **«вы»**, забота о теле, **без стыда и агрессивных обещаний**.
- Конкретика: для кого, в какой ситуации, что сделать; без воды.
- Товары: артикулы WB в формате как в колонке `текст` в `100постов_тг.csv` (например: «Артикул ВБ на … #…»).
- Дисклеймер по здоровью — коротко, если тема чувствительная.

## 4. Формат вывода: Markdown (обязательно)

Выдавай готовый пост **одним блоком Markdown**, чтобы его можно было скопировать в редактор или в Cursor.

### Подмножество для Telegram

Используй:

- Заголовок поста: строка с **одним эмодзи по смыслу** и коротким текстом (уровень `##` или жирный первый абзац — см. шаблон ниже).
- **Жирный** для 1–2 акцентов и подписи к CTA.
- Списки: `-` или нумерация `1.`
- Цитата опционально: `>`
- Разделитель между блоками при необходимости: строка `---`
- Эмодзи в списках — умеренно, как в базе постов.

Не используй в основном теле поста сложную вёрстку (таблицы HTML), если пользователь не просил отдельно.

### Шаблон поста (копируй структуру)

```markdown
## [Эмодзи] Короткий заголовок

Лид: 1–2 предложения — контекст и боль/надежда.

Основная мысль абзацем или короткими абзацами.

- пункт пользы или шага;
- ещё пункт;
- при необходимости ещё.

**Мягкий CTA:** например, сохраните / загляните в витрину / попробуйте сегодня.

---

**Товары (если уместно):** Артикул ВБ на [название] #XXXXXXXX

*Короткий дисклеймер только если нужен (здоровье, индивидуальная консультация врача и т.п.)*
```

### Вариант «только текст для карусели»

Если просят слайды — после основного поста добавь блок:

```markdown
### Слайды карусели (по желанию заказчика)

1. **Слайд 1:** …
2. **Слайд 2:** …
```

## 5. После текста поста (по запросу)

Если пользователь просит метаданные для публикации, добавь в конце отдельным блоком:

```markdown
---
**Мета:** рубрика — …; платформа — …; хэштеги — по желанию (не спамить).
```

## 6. Обязательный вопрос о сохранении поста

После того как пост написан, агент **обязательно** задаёт вопрос:

```markdown
Сохранить этот пост в базу?
- Если да — напишите: «Беру в работу пост».
- Если в черновик — напишите: «В черновик».
```

Дальше действуй строго по ответу:

- **«Беру в работу пост»** → сохранить пост отдельным `.md` в `КОНТЕНТ/ГОТОВЫЕ ПОСТЫ/`.
- **«В черновик»** → сохранить пост отдельным `.md` в `КОНТЕНТ/ПОСТЫ/` как черновик (чтобы можно было вернуться позже).
- Если пользователь не дал явный статус — уточни ещё раз и не выбирай статус сам.

Рекомендуемое имя файла:
- `YYYY-MM-DD_краткая_тема.md` (русские буквы допустимы).

## 7. Контроль перед ответом

- [ ] Если тема не была задана пользователем — тема выбрана самостоятельно, с учётом прошлых постов и рубрик; перед постом есть блок **Тема поста** / **Почему эта тема**
- [ ] После поста задан обязательный вопрос: сохранить в базу или в черновик
- [ ] Тон и запреты совпадают с `Карточка_бренда_для_СММ_агента.md`
- [ ] Стиль согласован с примерами из `100постов_тг.csv`
- [ ] Нет чужих брендов из файлов конкурентов
- [ ] Пост **полностью в Markdown**, структура как в шаблоне
